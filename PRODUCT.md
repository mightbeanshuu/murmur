# Product

## Register

product

## Users

Murmur serves technical builders, AI practitioners, recruiters, and hiring managers evaluating a production-shaped agent system. Builders need to turn a complex goal into a visible, inspectable swarm run; reviewers need to understand the product's technical credibility and interaction model quickly without reading the repository first.

## Product Purpose

Murmur turns one complex goal into a live, durable, validated agent swarm. It makes planning, delegation, execution, validation, and synthesis legible in a single workspace while demonstrating production-quality authentication, billing, orchestration, persistence, and observability. Success means a user can confidently launch a run, follow its progress, inspect agent output, understand system state, and manage their account without friction.

## Brand Personality

Precise, energetic, and technically credible. Murmur should feel like an advanced instrument with a distinctive neon pulse: confident enough for a portfolio showcase, restrained enough for repeated product use, and never apologetic about being an AI-native interface.

## Anti-references

Avoid generic hackathon dashboards, decorative sci-fi chrome, illegible low-contrast glass, card grids that obscure the core workflow, and neon effects applied without meaning. Avoid interfaces that require repository knowledge to understand the product, hide system state, or make standard account and billing actions feel experimental.

## Design Principles

1. Make the swarm legible: goal, plan, active work, dependencies, validation, and final output should read as one coherent flow.
2. Earn the spectacle: neon, glow, and glass may reinforce live computation and state, but the task remains visually dominant.
3. Prove depth progressively: the first screen is immediately understandable while deeper system, infrastructure, and agent details stay available on demand.
4. Keep account actions trustworthy: authentication, plans, payments, limits, errors, and success states use familiar language and predictable behavior.
5. Design for both use and evaluation: experienced users can work efficiently, while a recruiter or reviewer can grasp Murmur's value and engineering maturity in minutes.

## Accessibility & Inclusion

Target WCAG 2.2 AA contrast and keyboard access. Never communicate agent or infrastructure state by color alone. Respect reduced-motion preferences, preserve visible focus states, use readable labels and error messages, and keep the core workflow usable across desktop, tablet, and mobile layouts.
